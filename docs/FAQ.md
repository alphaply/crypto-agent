## <a id="faq"></a>❓ FAQ / 常见问题解答

### Q1: 我应该使用什么 LLM 模型？
LLM需要支持function call以及OpenAI的调用方式。可以直接在.env中配置使用。  
这意味着reasoning的model不兼容，因为程式要求第一个肯定是工具调用  
目前2026年2月4日19:28:09来看，越贵的模型表现越好。  
个人感受来看，oups4.6和gpt5.2感觉差不多，性价比来看国产的qwen3-max比较好。

### Q2: 会爆仓吗？
在实盘模式下，15m的监控力度，Agent一般不会爆仓，会自动去调用工具执行平仓命令。  
除非遇到极端行情的币种，建议配置市占率较大的币种进行测试。
在2026年2月1日的极端下跌行情中，qwen3-max尝试进行抄底，虽然进场位置不佳，但是会自行保本出局。未受影响。
人类的有所参与的只有清空历史分析思路（limit 3的那个）清空的主要目的就是让它开单，如果有历史记录在，在极端下跌行情中是不会开单的。
注意，虽然agent掌管了api，但是最大的因素还是人为进行控制，现在正在努力改善这一情况，使用越强的模型越不需要改善

### Q3: 建议使用多少仓位进行测试？
低杠杆，金额不建议超过100U进行测试

### Q4: 怎么运行项目，要域名服务器吗？
不需要域名 不需要服务器，如果是本地测试，大陆可能需要配置代理，在`agent_graph.py`中初始化**MarketTool**的时候传入**proxy_port**参数，或者在`market_data.py`中直接修改默认值。
可以不访问网页，只查看终端。

### Q5: 为什么需要清除历史记录？
在 Dashboard 网页端导航栏，点击 **"🗑️" (清除)** 图标。
* 系统会弹窗提示输入密码。
* 输入你在 `.env` 中配置的 `ADMIN_PASSWORD` 即可删除该币种的所有 **AI 分析历史 (Summaries)**。

在prompt当中有Agent的分析，思维历史记录，limit我设置的3条，在初始构想中1h的话limit 24,15m的limit 4。
建议三天清理一次，或者一天清理一次（在空仓的时候清理）

### Q7: Agent的输入有哪些？
Agent的详细输入可以配置langchain相关环境变量后去smith.langchain.com查看。
当前版本的Agent输入只有一些常规技术指标：RSI，EMA20 50 100 200 资金费率 持仓量 波动率 当前仓位 当前挂单，近3条的历史记录等等。

### Q8: 实盘Agent调用的工具输出是？
可以执行Buy Sell的限价单操作，至于为什么实盘的工具没有考虑止盈止损，是因为条件委托死活获取不到，导致Agent重复添加条件委托。  
CLOSE平仓指令可以平掉多空，默认的话是账户开启双向持仓。平空还是平多由Agent传入的参数决定。
CANCEL是Agent撤销挂单的指令。这里的风险就在于，Agent的一次执行耗时大概是20s++
行情极端可能会无法撤销订单,请自行承担此类风险。

### Q9：为什么我不推荐deepseek？
deepseek在我使用下来，感受就是，输出的内容很有逻辑性，但是一直都是小亏小亏小亏小亏的损单。开单时间半天没有一次盈的。  
如果有异议，可以在pr中给出具体的解决方案，非常感谢！

### Q10：Prompt需要我修改吗？
是的，每个人的做单思维不一样，我写的Prompt间接传达了我的做单方式与思维。需要自行斟酌修改成时候自己口味以及适合自己设置的llm的口味。
在prompts.py文件中修改Prompt。
