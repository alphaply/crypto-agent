# FAQ（v1.0）

## 1. 这个项目能保证盈利吗？

不能。它是自动化交易执行与分析系统，不是稳赚策略。请先模拟再实盘，并控制仓位。

## 2. v1.0 还支持 screener 双模型分流吗？

不支持。v1.0 已移除 screener 分流逻辑，统一为主决策执行链。

## 3. 支持哪些模式？

- `REAL`：实盘合约
- `STRATEGY`：模拟策略
- `SPOT_DCA`：现货定投

## 4. 本地就能跑吗？

可以。本地启动 `dashboard.py` 即可使用 UI 与调度能力。云服务器仅在你需要 24h 持续运行时才需要。

## 5. 模型应该怎么选？

要求支持 OpenAI 兼容接口与工具调用。实际建议：先选择稳定、低成本模型做策略验证，再逐步升级高性能模型。

## 6. 如何控制成本？

- 降低 `REAL/STRATEGY` 运行频率。
- 减少并行配置数量。
- 使用更低单价模型。
- 定期查看 Stats 页面中的 token 与成本统计。

## 7. 为什么定价改了重启后又回去？

v1.0 已修复：后台改动会同步回写 `pricing.json`。若仍异常，请检查文件写权限。

## 8. 为什么 History 页有时看不到数据？

常见原因：

- 当前 `symbol` 下没有对应 `config_id` 记录。
- 配置被禁用或刚创建尚未产生历史。
- 实盘模式需要先有 `balance_history` 数据。

## 9. `?symbol=` 空参数会怎样？

v1.0 会自动移除空参数并回退到默认标的，不再显示空标的状态。

## 10. Prompt 模板怎么新建？

在 Admin 的 Prompt 模板区输入名称后点击“新建 TXT”，会自动补全 `.txt` 并创建文件。

## 11. 数据库损坏或脏数据怎么处理？

先备份 `trading_data.db`，再按问题范围清理历史数据。不要直接在实盘运行时强删数据库。

## 12. 为什么有时下单失败？

重点排查：

- 交易所最小下单额限制（notional）。
- API 权限不足。
- 网络或时间同步问题。
- 杠杆、数量或价格精度不符合交易所规则。
