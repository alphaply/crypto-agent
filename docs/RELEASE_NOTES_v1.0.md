# Release Notes v1.0

发布日期：2026-04-10

## 概览

v1.0 是首个正式发布版本，目标是稳定、可维护、可运营。核心方向：

- 简化执行链路
- 提升前端可用性
- 强化成本与配置管理
- 完成文档体系重构

## 主要变更

### 1. 执行链路简化

- 移除 screener 分流节点及相关配置/展示逻辑。
- 统一主决策模型执行链，降低复杂度与维护成本。

### 2. History 与导航体验修复

- 修复 ALL 视图下多配置曲线空白问题（字段映射统一）。
- 修复 `?symbol=` 空参数行为，自动回退默认标的。
- 移除历史页“生成昨日汇总”手动按钮入口。
- 移除导航栏“清理”快捷入口。

### 3. Prompt 编辑体验升级

- Admin 页面 Prompt 编辑区升级为高级编辑器（行号、搜索、快捷保存）。
- 新建模板流程强化（PC 端更明确的新建入口与校验提示）。

### 4. 模型定价管理增强

- 新增模型定价删除能力（前后端全链路）。
- 定价变更后自动回写 `pricing.json`，重启不回流。
- `pricing.json` 加入 git ignore，避免本地定价泄露。

### 5. 文档体系重构

- 重写 README、安装、配置、成本、FAQ。
- 更新开发说明与仓库规范，去除过时内容。

## 兼容性说明

- 旧配置中的 `enable_screening` / `screener` 字段已废弃。
- 建议基于 `.env.template` 重新对齐配置字段。

## 升级建议

1. 备份 `.env` 与 `trading_data.db`。
2. 合并 v1.0 代码后，核对 `SYMBOL_CONFIGS` 字段。
3. 在 `STRATEGY` 模式完成一次回归后再启用 `REAL`。
4. 在 Admin 中检查模型定价是否符合你的供应商单价。

## 已知限制

- 自动化回测框架与系统化测试集仍在后续版本规划中。
- 文档中的成本示例为估算值，实际受模型与上下文长度影响。
