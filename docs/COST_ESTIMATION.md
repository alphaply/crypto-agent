# 成本估算与胜率测评 (Cost & Performance)

运行本交易 Agent 的成本主要来自 LLM 的 API 调用。本指南提供了基于最新版本的消耗参考以及胜率记录模版。

## 1. 运行成本分析

根据 V0.3 版本的实测，单次运行的 Token 消耗受市场波动、指标数量及上下文长度影响，通常在 **3k - 4k Tokens** 之间。

| 模式 | 运行频率 | 每日调用次数 | 预估单次 Token | 每日总 Token (预估) |
| :--- | :--- | :--- | :--- | :--- |
| **STRATEGY (模拟)** | 1 小时 / 次 | 24 次 | ~3.5k | ~84k |
| **REAL (实盘)** | 15 分钟 / 次 | 96 次 | ~3.5k | ~336k |

### 费用计算参考 (以 GPT-4o-mini 为例)
*当前价格假设: Input $0.15/1M, Output $0.60/1M*

- **STRATEGY 模式**: 约 `$0.02 / 天 / 交易对`
- **REAL 模式**: 约 `$0.10 / 天 / 交易对`

> **注意**: 以上为理想情况下的最低估算。如果你在 `SYMBOL_CONFIGS` 中开启了 `Summarizer` 或频繁与机器人进行 Chat 对话，成本会相应增加。
